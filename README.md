Expr1. Как поменять местами значения двух переменных? Можно ли это сделать без ещё одной временной переменной. Стоит ли так делать?
Способ 1:
Использовать временную переменную для «промежуточного переливания» одно переменной в другую. Например A, B, C:
C = A
A = B
B = C

Таким образом, мы поменяем значения A и B местами, используя временную переменную C.
Способ 2:
Без использования временной переменной. Использование выражений суммирования и вычитания:
A = A + B 
B = A - B
A = A – B
Например: A = 5, B = 10, тогда:
A = 5 + 10 = 15 (Теперь в A лежит 15, а в B всё еще лежит 10)
B = 15 – 10= 5 (Теперь в B лежит 5 – значение, которое принадлежало A)
A = 15 – 5 = 10 (Записали теперь значение 10 – изначальное значение переменной B, в переменную A).

Использовать второй способ не имеет смысла, т.к. подобные арифметические действия усложняют читаемость кода, и при работе с большими переменными, при приближении к предельным значениям типа int, double или float, может возникнуть проблема выхода за пределы допустимых значений данного типа, что может привести к «поломке» правильного числа. Первый способ является более простым, т.к. даже если вернуться к коду спустя время, он не потеряет в читаемости, и будет понятно, что в нашем случае C будет являться временной переменной, а также C будет хранить данные, которые мы ей передаем без арифметических вычислений.

Expr2. Задается натуральное трехзначное число (гарантируется, что трехзначное). Развернуть его, т.е. получить трехзначное число, записанное теми же цифрами в обратном порядке.
Способ 1:
Преобразование строки.
Получаем значение A, которое гарантируется, что трехзначное через “Console.ReadLine();”, разбивает его на переменные A1, A2, A3 как отдельные символы с использованием char. После чего, кортежем можно кортежем выполнить разворот:
(A1, A2, A3) = (A3, A2, A1)
После этого переводим строку в целые числа, перемножаем первые два значения на 100 и 10 и делаем сумму.
A1 = int.Parse (A1.ToString());
A2 = int.Parse(A2.ToString());
A3 = int.Parse (A3.ToString());
A = A1 * 100 + A2 * 10 + A3;
Способ 2:
Арифметически.
Представим A как число 357. Необходимы два оператора: “/”(деление) и “%”(остаток от деления). Чтобы получить последнюю цифру, необходимо применить оператор % (остаток от деления):
357 % 10 = 7; (A3 = A % 10 = 7)
Чтобы получить первую цифру, необходимо применить оператор / (деление):
357 / 100 = 3; (A1 = A / 100 = 3)
Чтобы получить среднюю цифру, можем применить комбинацию:
357 / 10 = 35 (A2 = A / 10 = 35)
35 % 10 = 5 (A2= A2 % 10 = 5)
Но также можно и наоборот (получить последние две цифры использовав оператор остатка “% 10”, получив при этом 57, и после этого «A2 / 10”.
После произвести сборку, используя формулу в конце Способ 1.

Expr3. Задано время Н часов (ровно). Вычислить угол в градусах между часовыми и минутными стрелками. Например, 5 часов -> 150 градусов, 20 часов -> 120 градусов. Не использовать циклы.

На часах есть 12 делений, которые составляют 360 градусов. Следовательно, при использовании 24-часового формата, необходимо отбросить лишние 12 часов(после полудня), чтобы получить 12-часовой формат.
360 / 12 = 30 градусов – один час.
Исходя из данных в задаче, минутная стрелка находится ровно на 12 часах, поэтому, самый простой алгоритм:
int H = int.Parse(Console.ReadLine()); //(получаем данные из консоли, делаем каст в тип int)
H = H % 12 (Если число больше 12, получим остаток, иначе – останется исходное число)
sC = H * 30 (т.к. 1 час = 30 градусам)
bC = 360 – sC (определяем второй угол)
C = Math.Min(sC, bC) (получаем наикратчайший угол между минутной и часовой стрелкой).
Например, H = 20:
H = 20 % 12 = 8;
sC = 8 * 30 = 240;
bC = 360 – 240 = 120;
C = Math.Min(240, 120) = 120;

Expr4. Найти количество чисел меньших N, которые имеют простые делители X или Y.
N является неким пределом чисел, определенных как (N – 1), т.к. N не входит в число искомых значений, а является лишь его пределом. Следовательно, из множества N-1 есть некие два числа X и Y, которые являются заданными числами, на которые делятся числа, которые меньше числа N. При этом все числа являются целочисленными значениями переменных.
По условиям задачи, X и Y должны быть простыми числами (2, 3, 5, 7, 11, 13, 17, 19 и т.д.)
Int N = int.Parse(Console.ReadLine()) (получение значения верхнего предела)
Int X = int.Parse(Console.ReadLine()) (получение значения X делителя)
Int Y = int.Parse(Console.ReadLine()) (получение значения Y делителя)
Получив данные от пользователя, подсчитываем простым делением:
countX = (N – 1) / X;
countY = (N – 1) / Y;
Чтобы избежать суммирования повторных значений, необходимо убрать возможные повторяющиеся значения, чтобы программа не посчитала их дважды:
countXY = (N – 1) / (X * Y);
Далее, суммируем количество чисел, делящиеся на X и на Y, вычитая XY:
countN = countX + countY – countXY;
Попробуем ввести значения: N = 12, X = 2, Y = 3, тогда:
countX = (12 – 1) / 2 = 5
countY = (12 – 1) / 3 = 3
countXY = (12 – 1) / (2 * 3) = 11 / 6 = 1
countN = 5 + 3 – 1 = 7 – количество чисел, которые можно поделить на 2 и на 3.

Expr5. Найти количество високосных лет на отрезке [a, b] не используя циклы.
Високосный год придерживается трех правил:
- Год должен делится на 4.
- Но если он делится на 100, то он не високосный.
- Исключение: если он делится на 400, то он всё равно високосный.
Отрезок [a, b] можно определить как (Количество от 1 до b) – (Количество от 1 до a-1)
следовательно, a и b – входные величины. Вычисления для нахождения B, которое мы подсчитываем как диапазон от 1 до b:
int B = int.Parse(Console.ReadLine());
Далее, следуя первому правилу:
int count4B= B / 4;
Далее:
int count100B = B / 100;
Затем:
int count400B = B / 400;
Далее получаем следующее выражение:
int countX = count4B – count100B + count400B;
 
Вычисления для нахождения количества для A. Так как мы учитываем A в отрезке [1, b], то отрезов [1, A] будем считать как [1, A – 1]:
int A = int.Parse(Console.ReadLine());
Далее, следуя первому правилу:
int count4A= (A - 1) / 4;
Далее:
int count100A = (A - 1) / 100;
Затем:
int count400A = (A - 1) / 400;
Далее получаем следующее выражение:
int countY = count4A – count100A + count400A;
Таким образом, получив количества високосных лет для каждого из промежутков, делаем финальные вычисления:
int countXY = countX – countY;

Expr6. Посчитать расстояние от точки до прямой, заданной двумя разными точками.
Для расчета расстояния от точки до прямой, заданной двумя разными точками, нам необходимо учесть 3 разные координаты:
-	Точка, от которой ищем расстояние (X0, Y0);
-	Первая точка на прямой: (X1, Y1);
-	Вторая точка на прямой: (X2, Y2);
Данные три точки можно представить как треугольник, следовательно,
C(X0, Y0), A(X1,Y1), B(X2,Y2) – треугольник. Расстояние от точки C(X0, Y0) до прямой – высота, опущенная из этой точки на основание a(A(X1,Y1), B(X2,Y2)).
Тогда, исходя из формулы нахождения площади, выразим высоту:
S=(1/2)*a*h
H=(2*S)/a
Исходя из формулы нахождения площади по координатам вершин(Формула Гаусса):
S = (1/2) * |(X2 – X1) * (Y1 – Y0) – (X1 – X0) * (Y2 – Y1)|, где | | - знак модуля, т.к. значение не может быть отрицательным.
Длину a можно выразить из формулы Пифагора для координат:
a(A, B) = sqrt( (X2 - X1)^2 +(Y2 – Y1)^2 ), где sqrt – квадратный корень.
Все числа будут иметь тип double для более точных вычислений.
Принимаем данные от пользователя. Заведомо известно, что пользователь не будет вводить одинаковые данные для точек, и выражение не даст 0:
 
// C:
double X0 = double.Parse(Console.ReadLine());
double Y0 = double.Parse(Console.ReadLine());
// A:
double X1 = double.Parse(Console.ReadLine());
double Y1 = double.Parse(Console.ReadLine());
// B:
double X2 = double.Parse(Console.ReadLine());
double Y2 = double.Parse(Console.ReadLine());

Используя входные данные находим площадь S и основание a(A, B):
double a = Math.Sqrt( Math.Pow((X2-X1), 2) + Math.Pow((Y2-Y1), 2) );
double S = (1.0/2) * Math.Abs( (X2-X1)*(Y1-Y0) - (X1-X0)*(Y2-Y1) );
double H = (2 * S)/a;
 
Expr7. Найти вектор, параллельный прямой. Перпендикулярный прямой. Прямая задана коэффициентами уравнения прямой.
Общее уравнение прямой на плоскости:
Ax+By+C = 0
Где A, B, C – входные данные коэффициентов(числа типа double)
A и B являются вектором нормали(перпендикуляром данной прямой).
Направляющий вектор s лежит на самой прямой (или параллелен ей), значит, он перпендикулярен вектору нормали.
Чтобы сделать один вектор перпендикулярным другому на плоскости, нужно поменять их координаты местами и у одной из них сменить знак на противоположный(чтобы их скалярное произведение стало равно нулю). Например: нормальный вектор (3, 5), то параллельный вектор (-5, 3) или (5, -3)
 
Отсюда следует:
//A(x):
double Ax = double.Parse(Console.ReadLine());
//B(y):
double By = double.Parse(Console.ReadLine());
=>
//Перпендикулярный вектор:
public static string GetPerpendicularVector(double Ax, double By)
{
	return Ax + “ “ + By;
}
// Параллельный вектор:
public static string GetPerallelVector(double Ax, double By)
{
	return By + “ “ + (-Ax);
}
ИЛИ
public static string GetPerallelVector(double Ax, double By)
{
	return (-By) + “ “ + Ax;
}
 
Expr8. Дана прямая L и точка A. Найти точку пересечения прямой L с перпендикулярной ей прямой P, проходящей через точку A. Можете считать, что прямая задана либо двумя точками, либо коэффициентами уравнения прямой — как вам удобнее.
Зададим прямую L двумя точками M1(X1, Y1) и  M2(X2, Y2), а точку, из которой опускаем перпендикуляр, назовем A(X0, Y0).
Вектор направляющий (вдоль прямой): v = (X2 – X1, Y2-Y1)
Вектор к нашей точке A: v = (X0 – X1, Y0 – Y1)
Точка пересечения P(Xp, Yp) лежит на прямой L. Для нахождения сдвиг относительно стартовой точки M1 используем скалярное произведение векторов.
Коэффициент сдвига k:
k = (u*v)/(v*v) = ( (X0-X1)*(X2-X1)+(Y0-Y1)*(Y2-Y1) ) / ((X2-X1)^2 + (Y2-Y1)^2)
Зная коэффициент k, координаты точки пересечения P находятся следующим образом:
Xp = X1 + k*(X2-X1)
Yp = Y1 + k*(Y2-Y1)

 
Ввод данных от пользователя:
// M1 (x1, y1):
double X1 = double.Parse(Console.ReadLine());
double Y1 = double.Parse(Console.ReadLine());
// M2 (x2, y2):
double X2 = double.Parse(Console.ReadLine());
double Y2 = double.Parse(Console.ReadLine());
// A (x0, y0):
double X0 = double.Parse(Console.ReadLine());
double Y0 = double.Parse(Console.ReadLine());
// Считаем вектор вдоль прямой:
double Vx = X2 – X1;
double Vy = Y2 – Y1;
// Считаем вектор к нашей точке:
double Ux = X0-X1;
double Uy = Y0-Y1;
Тогда, коэффициент сдвига:
double k = (Ux*Vx+Uy*Vy)/(Math.Pow(Vx, 2)+Math.Pow(Vy, 2));
Отсюда:
double Xp = X1 + k*Vx;
double Yp = Y1 + k*Vy;
return Xp + “ “ + Yp;
